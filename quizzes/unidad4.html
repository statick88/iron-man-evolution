35  Quiz 4: Refinamiento

35.1 Pregunta 1

¿Cuál es la verdad sobre el primer output de una IA?

  1. Siempre es perfecto
  2. Es solo el comienzo, no el final
  3. Nunca es útil
  4. Debe usarse tal cual
Respuesta B) Es solo el comienzo, no el final

35.2 Pregunta 2

¿Qué hace Stark cuando el primer código no es óptimo?

  1. Lo usa de todas formas
  2. Lo critica y pide mejora
  3. Elimina todo
  4. Pide otro código diferente
Respuesta B) Lo critica y pide mejora

35.3 Pregunta 3

¿Por qué Stark no pidió TODO en la primera iteración?

  1. No sabía lo que necesitaba
  2. Quería iterar para descubrir requisitos
  3. Quería perder tiempo
  4. No confía en la IA
Respuesta B) Quería iterar para descubrir requisitos

35.4 Pregunta 4

¿Qué revela el primer output sobre el contexto inicial?

  1. Que el contexto estaba perfecto
  2. Que faltaba algo
  3. Que la IA no sirve
  4. Que el código está bien
Respuesta B) Que faltaba algo

35.5 Pregunta 5

¿Cuántas versiones tomó Stark para la función de primos?

  1. 1
  2. 2
  3. 3
  4. Muchas
Respuesta C) 3

35.6 Pregunta 6

¿Por qué el sistema de logging evolucionó de simple a robusto?

  1. Porque J.A.R.V.I.S. lo hizo solo
  2. Porque cada iteración reveló nuevos requisitos
  3. Porque Stark cambió de opinión
  4. Porque era necesario
Respuesta B) Porque cada iteración reveló nuevos requisitos

35.7 Pregunta 7

¿Cuándo se considera “suficientemente bueno” un output?

  1. Nunca
  2. Cuando cumple los requisitos de uso
  3. Cuando es perfecto
  4. Cuando toma mucho tiempo
Respuesta B) Cuando cumple los requisitos de uso

35.8 Pregunta 8

¿Qué es “iteration is the process”?

  1. Que la perfección existe
  2. Que el primer intento es el último
  3. Que la mejora continua es normal y necesaria
  4. Que no hay que iterar
Respuesta C) Que la mejora continua es normal y necesaria
